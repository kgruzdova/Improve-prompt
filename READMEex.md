# Improve-prompt

Использование
Запустить скрипт и ввести первый запрос (например: «покажи весь список»).
Уточнять список: «добавь оплату за гитару 1200», «отметь, что ЖКХ оплачено», «удали кружки».
При запросе «текущий список» или «весь список» модель возвращает полный список с учётом статусов.
При новом запуске скрипта список восстанавливается из payment_state.json.
Завершить сессию Ctrl+C.

prompts_direct.json
Файл с заготовленным системным промптом для OpenAI. Содержит массив prompts, каждый элемент — объект с полями:

Поле	Описание
id	Идентификатор промпта
name	Название (например, «Напоминалка по оплатам»)
role	Роль модели (помощник по финансам, чек-листы-напоминалки)
context	Контекст: категории оплат, горизонт, язык ответа
task	Задача: составить чек-лист с сроками и каналами оплаты
format	Формат ответа: JSON с полями title, steps, notes. Инструкция: при запросах «текущий список», «весь список» — возвращать полный список, оплаченные пункты помечать как «— оплачено» и не удалять
Скрипт использует только первый промпт из массива.

openai_direct.py
CLI-скрипт для создания напоминалок по оплатам через OpenAI API.

Требования
Python 3.x
openai, python-dotenv
OPENAI_API_KEY в .env
Функциональность
Загрузка промпта — читает prompts_direct.json
Диалог с сохранением контекста — можно добавлять, удалять и отмечать оплаты
Сохранение списка — после каждого ответа чек-лист пишется в payment_state.json
Восстановление при запуске — при новой сессии загружается сохранённый список
Параметры запроса — настраиваемые temperature и max_tokens
Статистика — количество сообщений и токенов за сессию
Команды в чате
Команда	Действие
/помощь	Показать справку по командам
/очистить	Очистить историю диалога и сохранённый список
/история	Показать историю сообщений
/настройки	Изменить temperature и max_tokens
/выход	Завершить сессию
Файлы
Файл	Описание
prompts_direct.json	Системный промпт
payment_state.json	Сохранённый чек-лист (создаётся автоматически)
Запуск
python openai_direct.py
Структура кода
OpenAIClient — работа с OpenAI API, отправка сообщений и учёт токенов
load_prompt() — загрузка промпта из JSON
load_saved_state() — загрузка сохранённого списка из payment_state.json
save_state() — сохранение чек-листа в файл
try_parse_checklist() — извлечение JSON из ответа модели
build_system_message() — сборка системного сообщения (роль + контекст + задача + формат + сохранённое состояние)
get_settings() — ввод temperature и max_tokens
main() — интерактивный цикл диалога
Формат ответа модели
Модель возвращает JSON с полями:

title — заголовок чек-листа
steps — массив шагов (3–6 строк)
notes — массив тезисов (1–3 шт.)
